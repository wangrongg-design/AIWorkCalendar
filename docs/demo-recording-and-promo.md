# Work Calendar AI 演示录制与宣传方案

## 核心定位

一句话：用日历管理团队日报，用 AI 生成可用汇报。

首版宣传不要把产品讲成复杂项目管理系统，重点突出三个闭环：

- 员工用自然语言快速填日报和计划。
- 管理者用月历看团队填报率、缺失人数、风险和工时。
- AI 基于真实工作记录生成问答、分析和 Word 汇报。

## 录制流程

先启动本地演示服务：

```bash
pnpm demo:start
```

自动录制演示视频和截图：

```bash
pnpm record:demo
```

输出目录默认在 `tmp/demo-recordings/<时间戳>/`，其中：

- `work-calendar-ai-demo.webm` 是自动录制视频。
- `screenshots/` 是宣传页、日历、AI 问答、工作填报、智能汇报和企业管理截图。

如果想看着浏览器录制：

```bash
DEMO_HEADLESS=false pnpm record:demo
```

正式录制前建议重新执行 `pnpm demo:start`，让内存演示数据回到种子状态。不要在录制前调用 `/dev/clear-data`，它会清掉部门、项目和填报种子数据，影响演示完整度。

## 分镜脚本

1. 产品首页
   展示“用日历管理团队日报，用 AI 生成可用汇报”和免费试用入口。旁白强调企业免费试用 1 个月，试用期不限制成员人数并开放完整功能。

2. 登录进入日历看板
   手动输入演示账号 `admin@example.com / Passw0rd!`。旁白强调企业数据按租户隔离，管理员能看全公司视图。

3. 日历看板与 AI 对话
   展示填报率、未填报、风险数量，再点击 AI 对话询问“总结本月团队重点”。旁白强调回答只基于当前权限可见的日报和计划。

4. 工作填报
   点击新增填报，用自然语言生成日报草稿，再保存。旁白强调员工不需要按复杂模板手填。

5. 提交后 AI 分析
   提交工作填报，展示已提交状态、AI 标签和风险提示。旁白强调填报进入管理视图和后续汇报材料。

6. 智能汇报
   生成个人日报或周报，展示摘要、完成工作、风险问题、工时统计和 Word 下载入口。旁白强调从记录到汇报自动成稿。

7. 企业管理
   展示组织、订阅、启用成员计费和数据导出能力。旁白强调首版已经覆盖 1 个月不限人数试用、正式版按启用成员计费和基础合规。

## 30 秒口播

Work Calendar AI 是面向企业团队的 AI 工作填报和智能汇报工具。员工可以用自然语言快速生成日报和未来计划，管理者在月历里查看每天谁已填、谁未填、有哪些风险和工时投入。系统还能基于真实工作记录进行 AI 问答，并自动生成日报、周报和 Word 汇报。第一版支持企业 1 个月不限人数试用、组织权限、按启用成员计费、数据导出和审计记录，适合先从团队日报和管理汇报场景落地。
