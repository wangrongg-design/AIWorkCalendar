# AIWorkCalendar VI 手册 v1.0

## 设计定位

Work Calendar AI 是专业、可信、清晰、高效的企业级 AI 工作系统。视觉表达应服务高频使用、风险可见和行动明确，避免炫技型科技风、过度装饰和传统 OA 视觉。

关键词：企业级、数据可信、AI 辅助决策、低干扰、高频使用、风险可见、行动明确。

## 颜色系统

主色 Primary Blue 用于主按钮、导航选中态、关键链接和系统重点：

- `Primary 600`: `#0B57D0`
- `Primary 500`: `#1A73E8`
- `Primary 100`: `#D3E3FD`
- `Primary 50`: `#EEF5FF`

AI 辅助色 Teal 用于 AI 洞察、安全、智能建议和数据保密提示：

- `AI 600`: `#0F766E`
- `AI 500`: `#14A39A`
- `AI 100`: `#CCFBF1`
- `AI 50`: `#ECFDF9`

语义色：

- 完成/正常/通过：`#16A34A`
- 未填报/临近截止/提醒：`#D97706`
- 风险/阻塞/失败/删除：`#DC2626`

中性色：

- 主标题：`#111827`
- 正文重点：`#1F2937`
- 正文：`#374151`
- 次级文字：`#6B7280`
- 占位符：`#9CA3AF`
- 分割线：`#E5E7EB`

- 浅容器：`#F3F6FA`
- 卡片：`#FFFFFF`

## 使用原则

- 一个页面最多一个强主按钮。
- 蓝色只用于可行动、当前选中和系统重点。
- 青绿色只用于 AI、洞察、安全和辅助智能，不用于普通主按钮。
- 红色只表示真实风险或破坏性操作。
- 状态不能只靠颜色表达，必须配文字或图标。
- 正文对比度不低于 4.5:1。

## 页面结构

后台页面统一结构：

1. 页面标题区
2. 工具栏 / 筛选区
3. 核心数据区
4. 详情 / 表格 / 日历 / 列表

页面标题：`28px / 36px / 600`。副标题：`14px / 22px / #6B7280`。

工具栏：背景 `#F3F6FA`，圆角 `16px`，内边距 `12px`，控件高度 `40px`。

卡片：背景 `#FFFFFF`，边框 `#E5E7EB`，圆角 `16px`，阴影轻微。

## 组件规范

- 主按钮：Primary Blue 实心。
- 次按钮：白底 + 灰边框。
- 危险按钮：红色描边或红色实心。
- AI 按钮：青绿色图标 + 文案，避免和主按钮竞争。
- 按钮/输入框圆角：`10-12px`。
- 卡片圆角：`16px`。
- 弹窗圆角：`20px`。
- Tag 圆角：`999px`。
- 表格表头 `12px` 中性灰，正文 `14px`，行高 `56-72px`。
- 编辑用 Modal，详情用 Drawer，AI 助手用右侧 Drawer，删除用 Popconfirm。

## AI 模块

AI 模块必须输出可行动信息，统一结构：

1. AI 标题
2. 一句话结论
3. 关键证据
4. 建议动作

风险内容必须转为 Warning / Danger，不继续使用 AI 色。

## 日历状态

- 完成：`#16A34A`
- 部分完成：`#1A73E8`
- 未完成：`#D97706`
- 风险：`#DC2626`
- 未来计划：`#14A39A`
- 无数据：`#D1D5DB`

今日使用蓝色浅底 + 蓝色边框；风险日期使用红色角标；未填报使用橙色点或条；未来计划使用青绿色点。关键风险必须有文字标签。

4. 统一登录页、首页、H5、运维端的品牌语言。
